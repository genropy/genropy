groupletChunk

An HTML preview of a Bag with a pencil button that opens a full edit dialog.

When to use

You have a form where most fields are simple textboxes, dates, selects. But two or three sub‑records — the territorio block of a comune, the anagrafica details, a stack of technical specs — deserve their own treatment. Inlining them as another formlet bloats the page; hiding them behind a tab buries them. groupletChunk sits in the middle: a compact read‑only HTML preview of the sub‑Bag in the form, with a pencil button at the bottom‑right that pops a modal containing the real editor.

It saves vertical space, reads better at a glance, and the edit affordance is always one click away. The dialog is built lazily — you pay nothing until the user clicks the pencil.

Quick example with handler

Adapted from 03_grouplet_chunk.py. The chunk lives inside a frameForm bound to glbl.comune; the editable fields stay in the formlet, the territorio sub‑block goes into the chunk.

from gnr.core.gnrdecorator import public_method


class GnrCustomWebPage(object):
    py_requires = "gnrcomponents/grouplet/grouplet:GroupletHandler"

    def main(self, root, **kwargs):
        form = root.frameForm(frameCode='comune_form',
                              datapath='.form', pkeyPath='.comune_pkey',
                              _anchor=True)
        form.formStore(table='glbl.comune', storeType='Item',
                       handler='recordCluster')
        fb = form.center.contentPane(datapath='.record').formlet(
            cols=2, table='glbl.comune')
        fb.field('denominazione', colspan=2)
        fb.field('sigla_provincia')
        fb.field('codice_comune')
        chunk_pane = fb.div(colspan=2, lbl='Territorio', height='60px')
        chunk_pane.groupletChunk(
            value='^#FORM.record',
            template="""
            <div style="color:#555;">
                ${<span>Zona altimetrica: $zona_altimetrica</span>}
                ${<span style="margin-left:8px;">Alt. $altitudine m</span>}
            </div>
            ${<div style="color:#555;">Comune montano: $comune_montano</div>}
            """,
            name='edit_territorio',
            handler=self.grp_territorio,
            title='Edit Territorio')

    @public_method
    def grp_territorio(self, pane, **kwargs):
        fb = pane.formlet(cols=2, table='glbl.comune')
        fb.field('zona_altimetrica')
        fb.field('altitudine')
        fb.field('comune_montano')

The preview renders the template with the current record values. Click the pencil — a modal opens with the three editable fields from grp_territorio. Edits commit back into the same Bag node, the template refreshes on close, the surrounding form sees the changes like any other field write.

Template syntax

The template is plain HTML interpolated by the templateChunk JS widget. Two markers exist:

  • $field_name — substitutes the value of field_name from the bound Bag. Plain text substitution, no escaping flag, no formatting hooks.
  • ${<wrapper>...$field...</wrapper>} — conditional block: the entire wrapper is rendered only if the $field inside has a non‑empty value. Use this to hide labels and separators when the underlying field is empty, instead of dangling colons and orphan units.

Everything else is standard HTML — inline styles, nested elements, whatever the documentation site allows. Keep it compact; the chunk is a preview, not a document.

With a resource

Same widget, but the dialog content comes from a grouplet resource instead of an inline handler. The resource + table pair points at the same files pane.grouplet(...) would consume.

chunk_pane.groupletChunk(
    value='^#FORM.record',
    template="""
    <div style="font-weight:bold;">$denominazione</div>
    <div style="color:#555;">$sigla_provincia - $codice_comune</div>
    """,
    name='edit_anagrafica',
    resource='anagrafica',
    table='glbl.comune',
    title='Edit Anagrafica')

The resource file lives at tables/comune/grouplets/anagrafica.py in the glbl package; the dialog wraps its grouplet_main exactly as a standalone grouplet would.

Auto‑discovered template from __info__

If the resource's __info__() returns a template (and optionally template_virtual_columns), you can omit both template= and virtual_columns= from the call site. The widget calls gr_getTemplatePars() server‑side and fills them in for you.

chunk_pane.groupletChunk(
    value='^#FORM.record',
    name='edit_codici_auto',
    resource='codici',
    table='glbl.comune',
    title='Edit Codici')

The resource declares its own preview:

class Grouplet(object):
    def __info__(self):
        return dict(
            caption='Codici',
            template='<b>$sigla_provincia</b> - $codice_comune',
            template_virtual_columns='sigla_provincia,codice_comune')

    def grouplet_main(self, pane, **kwargs):
        fb = pane.formlet(cols=2, table='glbl.comune')
        fb.field('sigla_provincia')
        fb.field('codice_comune')

Now every caller of resource='codici' gets the same preview without copying the template string around.

Topic as chunk

When resource points at a folder (a topic, e.g. territorio containing several child grouplets), the dialog renders a CSS grid of all the child grouplets instead of a single one. The grid layout is controlled with the same remote_grid_* kwargs used by the standalone grouplet widget when consuming a topic.

chunk_pane.groupletChunk(
    value='^#FORM.record',
    template="""<div style="color:#555;">
        ${<span>Zona: $zona_altimetrica</span>}
        ${<span> | Alt. $altitudine m</span>}
    </div>""",
    name='edit_territorio_topic',
    resource='territorio',
    table='glbl.comune',
    title='Territorio',
    remote_grid_columns=2,
    remote_grid_collapsible=True)

The preview is still one compact box on the form; the dialog opens onto a two‑column grid of collapsible cells, one per child grouplet.

Styling the preview box

Any box_* kwarg is forwarded to the outer div wrapping the template. Useful for a quick visual frame without touching CSS.

chunk_pane.groupletChunk(
    value='^#FORM.record',
    template='<div>$sigla_provincia - $codice_comune</div>',
    name='edit_dettagli',
    handler=self.grp_dettagli,
    box_padding='5px',
    box_background='#f9f9f9',
    box_border_radius='4px')

Parameters

ParamTypeDefaultDescription
valuestrrequired Data binding path. Must use absolute ^#FORM.record.field paths, not relative .field — the template is rendered outside the form's datapath scope.
templatestrNone HTML with $field placeholders and ${...} conditional blocks. Optional when resource provides one via __info__.
namestrrequired Unique dialog identifier. Passed to genro.dlg.memoryDataEditor as the editor key; collisions across the page reuse the same dialog instance.
handlercallableNone Method that fills the edit dialog. Mutually exclusive with resource; one of the two is required.
resourcestrNone Grouplet resource path. Points at a single grouplet file or a topic folder; the dialog content adapts accordingly.
tablestrNone For table‑scoped resources (tables/<tbl>/grouplets/...). Same semantics as on the standalone grouplet widget.
titlestrNone Dialog title shown in the modal header.
virtual_columnsstrNone Comma‑separated fields referenced by the template but not loaded by the main form. Auto‑filled from __info__.template_virtual_columns when present.
box_*any— Kwargs forwarded to the preview box container. E.g. box_padding, box_background, box_border_radius.
grid_columnsintNone Shorthand for remote_grid_columns when resource is a topic. Convenience alias.
remote_grid_*any— Grid layout params used by the dialog when resource is a topic: remote_grid_columns, remote_grid_collapsible, remote_grid_template_columns, remote_grid_gap.

Gotchas

Value path differs from grouplet

groupletChunk uses absolute ^#FORM.record.field paths because the template chunk is rendered separately from the surrounding form's datapath context. Relative .field paths that work on pane.grouplet(value='.sub') will break here — the template will substitute empty strings and the dialog will write to the wrong node.

grid_* is a shorthand alias

When resource points at a topic, grid_columns=2 is rewritten internally to grouplet_remote_grid_columns=2 before being handed to the dialog's grouplet. Both forms work, but they collide: passing both grid_columns=2 and remote_grid_columns=3 on the same call yields undefined precedence. Pick one.

The name is global to the page

It's used as the key of the memory data editor instance; two chunks with the same name will share the dialog and stomp on each other's edits. Use distinct names per chunk (edit_territorio, edit_anagrafica, ...), not generic strings like edit.
