groupletWizard

A step‑by‑step wizard where each step is a grouplet from a topic folder, with a numbered stepper bar and forward/backward navigation.

When to use

The form is long enough that dropping it on the user as a single page would be cruel. Onboarding flows, booking funnels, multi‑section data entry, regulatory questionnaires — anything that naturally splits into stages where each stage has its own focus and its own validation cycle. The user fills in one chunk, the wizard validates and saves it, then the next chunk appears.

It is the close sibling of groupletPanel, but with the opposite intent. The panel hands the user a menu and lets them roam between grouplets in whatever order they like. The wizard locks the order: step 2 is unreachable until step 1 validates, the stepper bar reports progress, and a single forward button drives the whole sequence. Reach for the wizard when the order of input matters — either because later steps depend on earlier values, or because you want a clear sense of completion at the end.

How it looks

A frame with three regions. The top is a numbered stepper bar: a row of circles connected by short lines, one per grouplet in the topic folder. The center is a GroupletForm showing the current step's grouplet, bound to the same value Bag throughout. The bottom is a single forward button whose label changes to completeLabel when the user reaches the last step.

Completed steps show a checkmark and a green connector behind them. The active step is enlarged with the accent color. Pending steps are dimmed. The label of the forward button is the caption of the next step, or completeLabel when there is no next step.

Quick example

Straight from 07_grouplet_nested_topics.py, test 3. A wizard pointed at the submethodtesting topic folder: every grouplet in that folder becomes a step.

pane.borderContainer(height='500px', border='1px solid silver',
                     datapath='.nested_wizard').groupletWizard(
    topic='submethodtesting',
    value='^.data',
    frameCode='nested_wizard',
    region='center')

The wizard reads the topic folder via gr_getGroupletMenu, builds one step per grouplet, and binds every step's GroupletForm to ^.data. The user sees a single Bag accumulating values as they step through.

Inside a Table Handler form

This is where the wizard earns its keep. Drop it inside th_form and you get a multi‑step form for a real database record, with the parent TH form auto‑saving when the user reaches the end. The booking demo in test/resources/tables/booking/th_booking.py is the canonical example: four grouplets — guest, stay, services, payment — one per stage of the reservation.

from gnr.web.gnrbaseclasses import BaseComponent


class Form(BaseComponent):
    py_requires = 'gnrcomponents/grouplet/grouplet:GroupletHandler'

    def th_form(self, form):
        form.center.contentPane().groupletWizard(
            table='test.booking',
            value='^#FORM.record',
            frameCode='booking_wizard',
            completeLabel='Confirm Booking',
            saveMainFormOnComplete=True,
            grouplet_remote_num_guests="=#FORM.record.num_guests")

    def th_options(self):
        return dict(dialog_height='300px', dialog_width='500px',
                    showtoolbar=False)

Three things are happening here. First, value='^#FORM.record' binds every step's GroupletForm to the parent TH record — each step writes into the same Bag the outer form will save. Second, saveMainFormOnComplete=True wires the wizard's completion event to form.save({destPkey:'*dismiss*'}): when the user clicks the last‑step button labelled Confirm Booking, the TH form persists the record and dismisses the dialog. Third, anything you pass with the grouplet_ prefix — here grouplet_remote_num_guests — is forwarded as a regular kwarg into the inner GroupletForm, useful for propagating context variables that individual step grouplets read.

Nested topics

A topic folder can contain sub‑folders, and the wizard flattens that hierarchy into a sensible step sequence. Top‑level items become individual steps. A sub‑topic becomes a single step that renders all of its child grouplets at once, laid out as a CSS grid via gr_loadGrouplet. The submethodtesting topic from 07_grouplet_nested_topics.py illustrates: alfa and beta are individual steps, then the third step is the inner sub‑topic showing gamma and delta side by side.

# test/resources/tables/.../grouplets/submethodtesting/
#   __info__.py
#   alfa.py        ← step 1
#   beta.py        ← step 2
#   inner/         ← step 3 (renders gamma + delta in a grid)
#     gamma.py
#     delta.py

This is the trick for keeping the stepper bar readable when you have a dozen grouplets: cluster the closely related ones in a sub‑topic and they collapse into a single “mixed” step.

Step validation

Before advancing, the wizard calls form.isValid() on the current step's GroupletForm. If validation fails, a floating_message publishes — Please complete required fields — and the step index does not advance; the form stays where it is with its errors. If validation passes the form saves, the frameCode_step_complete event publishes with the step code, and the stepper moves forward by one. On the last step the same path publishes frameCode_complete, which is what saveMainFormOnComplete listens to. Three JS bridge entry points are available for programmatic navigation: gnr_grouplet.wizardNext(node, frameCode), gnr_grouplet.wizardGoTo(node, targetIdx, frameCode), gnr_grouplet.wizardUpdateStep(node, idx, completeLabel, frameCode).

Parameters

ParamTypeDefaultDescription
valuestrrequiredDatapath the wizard's GroupletForm is bound to. Same Bag is used across all steps.
topicstrrequiredTopic name. Every grouplet found in grouplets/<topic>/ becomes a step, in __info__.priority order.
tablestrNoneOptional table reference. When set the grouplets are looked up under tables/<tbl>/grouplets/<topic>/.
frameCodestr'grplt_wizard'Frame identifier. Drives the published events (<frameCode>_complete, <frameCode>_step_complete) and the inner step form id.
completeLabelstr'Confirm'Label shown on the forward button when the last step is reached.
closeLabelstr'Close'Label of the dismiss button on the summary page (only when a summary template is defined in __info__).
saveMainFormOnCompleteboolFalseWhen True, the parent th_form is saved (and dismissed) on wizard completion.
grouplets_rootstr'grouplets'Override the root folder name when grouplets live somewhere other than grouplets/.
grouplet_*any—Any kwarg prefixed with grouplet_ is stripped of its prefix and forwarded to the inner GroupletForm.

Gotchas

Topic is required

Without topic= there is no step source — the wizard has nothing to enumerate. Unlike groupletPanel, which can offer the whole grouplet tree of a table when no topic is set, the wizard refuses to operate without an explicit topic folder. If you want a wizard over an entire table, make a topic folder for it and place the relevant grouplets inside.

Step order follows __info__.priority

If your steps appear in the wrong order, the fix is in each grouplet's __info__(): set priority=<int>, lower numbers come first. Filenames are not a sort key — the menu builder reads priorities. Missing priorities sort after the explicit ones, by an internal tie‑break that you should not rely on.

Validation blocks the forward button, not the stepper

wizardNext calls form.isValid() and refuses to advance. But wizardGoTo — the action wired on every numbered circle in the stepper bar — saves the current form unconditionally before jumping. If you let users click backwards in the stepper, an invalid current step is silently saved on its way out. Constrain the stepper buttons if you need stricter behavior.
