Topic Grid

When you point a grouplet at a folder instead of a single file, all child grouplets render as a CSS grid.

When to use

You have a record with a sprawling set of sub-fields that naturally cluster into themes: technical specs, anagrafica, commercial terms, territorio. Each cluster is already implemented as a grouplet that edits a fragment of the record's extra_data Bag. Stacking them by hand in a form means writing a layout for every screen and re-doing it when a cluster gets added.

Instead, you drop the grouplets into a folder — that folder becomes a topic — and you point a single grouplet widget at the folder. The engine walks the folder, builds a menu of children, and lays them out as a CSS grid. Each cell carries the child's caption as a header and its content below.

Folder structure of a topic

A topic is a directory under your table's grouplets/ folder (or under the page-level resources/grouplets/). Children are individual grouplet Python files; sub-directories are nested sub-topics that recurse. An optional __info__.py per folder carries topic metadata.

grouplets/
  technical/
    __info__.py
    environment.py
    dependencies.py
    inner/             # nested sub-topic
      __info__.py
      gamma.py
      delta.py

__info__.py is optional and carries metadata such as caption, template, virtual_columns, and priority. Files whose name starts with __ are skipped during menu building.

Basic topic grid

Point resource at a folder name. The engine resolves it under resources/tables/test.myticket/grouplets/technical/, lists the children, and lays them out in a two-column grid.

pane.grouplet(
    resource='technical',
    table='test.myticket',
    value='^.data',
    remote_grid_columns=2)

Each cell is an expandbox with the child grouplet's caption as the header and its content rendered underneath. Datapath for each cell is anchored to the child's label, so a environment.py grouplet writes into .data.environment.

Collapsible cells

Add remote_grid_collapsible=True and each cell header becomes a click target that expands or collapses its content. Useful when individual grouplets are tall and the user wants to focus on one at a time.

pane.grouplet(
    resource='technical',
    table='test.myticket',
    value='^.data',
    remote_grid_collapsible=True)

By default cells are collapsible when columns > 1 and locked open in a single column. Pass remote_grid_collapsible='closed' to start the cells closed.

Custom grid template

For uneven columns, bypass remote_grid_columns and supply your own grid-template-columns. Tune the gap between cells with remote_grid_gap.

pane.grouplet(
    resource='commercial',
    table='test.myticket',
    value='^.data',
    remote_grid_template_columns='1fr 2fr',
    remote_grid_gap='16px')

Without a table (page-level)

Omit table to resolve the folder under resources/grouplets/app/ at the page level. Same grid mechanics, different lookup root.

pane.grouplet(
    resource='app',
    value='^.record',
    remote_grid_columns=3)

Nested sub-topics

A sub-directory inside a topic becomes a nested topic and renders as a grid inside the parent cell. Recursion is unbounded. Example: a territorio topic that holds an altimetria sub-topic.

grouplets/
  territorio/
    __info__.py
    zona.py
    altimetria/
      __info__.py
      quota.py
      pendenza.py

pane.grouplet(
    resource='territorio',
    table='glbl.comune',
    value='^.data',
    remote_grid_columns=2,
    remote_grid_collapsible=True)

The nested cell carries the sub-topic caption and its content is the inner grid. Nested cells render in a minimal visual style (lighter chrome) to differentiate from top-level cells. The datapath for the nested grid is derived from the sub-topic name, so a child of altimetria writes into .data.territorio.altimetria.quota.

__info__.py

Optional per-folder metadata file. The engine looks for a Grouplet or GroupletTopic class with an __info__ classmethod, or a plain module-level info dict. Recognized keys:

  • caption — header label for the cell. Falls back to the file/folder name.
  • priority — numeric sort order within the topic. Menus are sorted by priority, caption.
  • template — pre-baked HTML template consumed by groupletChunk when this grouplet is wrapped in a chunk dialog.
  • virtual_columns — comma-separated columns the template needs but that are not in the main form, so the chunk can hydrate them.
  • tags / permissions — gate the grouplet by user tag set or table permission. A failing check hides the cell entirely.
  • is_enabled(page) — module-level callback that returns False to hide the grouplet dynamically.

Parameters (remote_grid_*)

The remote_grid_* prefix is stripped server-side and the remaining suffix is passed to _loadGroupletTopic. All parameters are optional.

ParamTypeDefaultDescription
remote_grid_columnsint1Number of equal-width columns in the grid.
remote_grid_template_columnsstrderived from columnsRaw CSS grid-template-columns; overrides columns. Example: '1fr 2fr'.
remote_grid_collapsiblebool / strauto*Click-to-collapse cells. Auto True when columns > 1, False in single column. Pass 'closed' to start collapsed.
remote_grid_gapstr'10px'CSS gap between cells.
remote_grid_directionstrauto'rows' or 'columns'. Defaults to 'columns' when columns > 1, otherwise 'rows'. Controls how cells flow across the grid axes.

Used by

Topic grids are not a separate widget. They are a rendering mode of the base grouplet widget, triggered when resource resolves to a folder rather than a single Python file. The same _loadGroupletTopic machinery is reused by groupletChunk (when the chunk's dialog points to a topic instead of a single grouplet), by groupletPanel in multiButton mode (the bar selects which topic-child to load), and by groupletWizard when a wizard step is itself a sub-topic — the step body renders as a nested grid of its children.

Gotchas

Resource resolution: file vs folder

If you have both technical.py and a technical/ folder under the same grouplets/ root, the file wins and you get a single grouplet rather than a topic grid. The resource loader builds a Bag where leaf nodes (files) take precedence over branch nodes (folders) with the same label. Rename one of the two: drop the .py file or give the folder a distinct name.

Skipped names

Children whose name starts with __ are filtered out of the menu (that is how __info__.py and __pycache__ stay invisible). Do not start a real grouplet file with a double underscore or it will silently disappear from the grid.

Sort order is priority then caption

Without priority in __info__, the menu sorts by caption alphabetically. If you want a specific cell order, set priority on each child — a missing priority defaults to 0, which means a single child with priority=5 can end up after several priority-zero siblings sorted by caption.
