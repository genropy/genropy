Quick Start

A working grouplet is a webpage class, one pane.grouplet(...) call, and a method that drops form fields into the box it receives.

Prerequisites

  • A standard GenroPy webpage scaffolding. The page lives at webpages/index.py (mandatory entry point) or any sibling page; you need the usual GnrCustomWebPage class with a main(self, root, ...) method.
  • The page mixes in the grouplet machinery via py_requires = "gnrcomponents/grouplet/grouplet:GroupletHandler" (you can chain other components with commas, as usual).
  • Nothing else to import: GroupletHandler declares css_requires and js_requires on itself, so the grouplet CSS and JS are pulled in automatically the moment the component is mixed in.

Minimal example — handler

The fastest path: a handler method on the same webpage. Drop a grouplet, point it at a Bag path, give it a callable. The callable receives a pane already rooted at that Bag node.

from gnr.core.gnrdecorator import public_method


class GnrCustomWebPage(object):
    py_requires = "gnrcomponents/grouplet/grouplet:GroupletHandler"

    def main(self, root, **kwargs):
        root.grouplet(value='^.data', handler=self.grp_address)

    @public_method
    def grp_address(self, pane, **kwargs):
        fb = pane.formlet(cols=1, border_spacing='3px')
        fb.textbox(value='^.street', lbl='Street')
        fb.textbox(value='^.city', lbl='City')
        fb.textbox(value='^.zip', lbl='ZIP')

Open the page, see the three fields, edit them. The Bag at .data updates live as you type — street, city, zip become children of that node, no glue code in between.

Minimal example — resource

Same screen, same behaviour, but the field layout moves out of the page into a reusable resource. The webpage stops caring what the fields are:

class GnrCustomWebPage(object):
    py_requires = "gnrcomponents/grouplet/grouplet:GroupletHandler"

    def main(self, root, **kwargs):
        root.grouplet(value='^.data', resource='examples/address_fields')

The resource file lives at projects/gnrcore/packages/test/resources/grouplets/examples/address_fields.py and looks like this:

class Grouplet(object):
    def __info__(self):
        return dict(caption='Address Fields', priority=1)

    def grouplet_main(self, pane, **kwargs):
        fb = pane.formlet(cols=1, border_spacing='3px')
        fb.textbox(value='^.street', lbl='Street')
        fb.textbox(value='^.city', lbl='City')
        fb.textbox(value='^.zip', lbl='ZIP')
        fb.textbox(value='^.country', lbl='Country')

The conventions: the file lives under a resources/grouplets/<path>.py folder of any package, the class is named Grouplet, and the entry method is grouplet_main. The resource string is the path relative to the grouplets/ root, without the .py suffix.

Two ways, one mental model

Handler or resource — same thing

The grouplet engine doesn't care which form you used. It hands a pane already datapath‑rooted to your code; your code drops widgets in. Handler is faster when the template is page‑specific. Resource is mandatory when the same field block must be shared across pages, or when you want it picked up automatically per table via the table= argument.

What you just unlocked

Add fields

Anything formlet accepts works: fb.textbox, fb.numberTextBox, fb.dateTextBox, fb.filteringSelect. The grouplet is just a wrapper around a contentPane at a datapath.

Bind to a record

Inside a TH form, point value at the form's record Bag: pane.grouplet(value='^#FORM.record', handler=self.grp_address). Edits flow into the record like any other form field.

Multiple grouplets

Drop several pane.grouplet(value=...) calls side by side, each on its own Bag path. They are independent boxes, edited in parallel, persisted independently.

Switch by record type

The resource argument accepts a datapath: resource='^.record_type'. The grouplet re‑renders whenever that value changes, swapping the entire field layout to match.

Where to go next

To add edit dialogs (open the grouplet content in a modal, commit on OK) use groupletChunk. For repeating sub‑records in a tabular layout use groupletGrid. For multi‑step forms with a progress bar use groupletWizard. For navigation between sections (tree + content, or button bar + content) use groupletPanel.
