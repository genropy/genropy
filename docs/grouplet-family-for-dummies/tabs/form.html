groupletForm

A grouplet wrapped in a form with its own store — load, save, validate independently from the surrounding page.

When to use

The base grouplet is a UI fragment bound to a Bag: it renders fields, but knows nothing about loading from a record, saving back, or tracking dirty state. groupletForm stacks a form lifecycle on top, so the embedded grouplet becomes an editing unit you can drive on its own.

Reach for it when an editor needs to live inline in the page but stay independent from whatever else is around it — for example a memory-mode editor for an in-page Bag (a small address block, a configuration node), or a DB record editor wired to an external record selector. It is also the building block used internally by groupletPanel and groupletWizard: each panel section and each wizard step is a groupletForm.

Memory store with handler

Pure in-memory editing of a Bag that already lives in the page datastore. The form has no table, no record, no save round-trip: it loads the Bag pointed to by value, hands it to the inner grouplet, and writes changes back through the same datapath.

def test_1_groupletform_memory_handler(self, pane):
    pane.data('.address_data', Bag(dict(
        street='Via Roma 1', city='Milano',
        zip='20100', country='Italia')))
    frame = pane.framePane(frameCode='mem_gf',
                            height='400px', border='1px solid silver',
                            datapath='.mem_groupletform')
    bar = frame.top.slotToolbar('5,lbl,*')
    bar.lbl.div('In-memory address editor', font_weight='bold')
    frame.center.contentPane(padding='10px').groupletform(
        loadOnBuilt=True,
        handler=self.grp_address, value='^.#parent.address_data',
        formId='mem_gf_form')

@public_method
def grp_address(self, pane, **kwargs):
    fb = pane.formlet(cols=2, border_spacing='6px')
    fb.textbox(value='^.street', lbl='Street', colspan=2, width='100%')
    fb.textbox(value='^.city', lbl='City')
    fb.textbox(value='^.zip', lbl='ZIP', width='6em')
    fb.textbox(value='^.country', lbl='Country')

loadOnBuilt=True tells the widget to call this.form.load() as soon as the grouplet is built on the client, so the editor shows up populated without an explicit trigger. formId is the public handle for the form: it is the key you pass to genro.formById() from anywhere in the page to drive load/save programmatically. Keep it unique, give it a meaningful name — the auto-generated fallback works but is unreadable.

Record store with handler

Same widget, different store. With store_handler='record' the form talks to a DB table: it loads a record by primary key, hands the record Bag to the inner grouplet, and saves back via the standard form save flow. The typical layout is a selector at the top (a dbselect in a toolbar) and the inline editor below.

def test_2_groupletform_record_handler(self, pane):
    frame = pane.framePane(frameCode='rec_gf',
                            height='400px', border='1px solid silver',
                            datapath='.rec_groupletform')
    bar = frame.top.slotToolbar('5,selector,*')
    fb = bar.selector.formbuilder(cols=1, border_spacing='1px')
    fb.dbselect(value='^.comune_pkey', dbtable='glbl.comune', lbl='Comune')
    frame.center.contentPane(padding='10px').groupletform(
        handler=self.grp_comune_fields, value='^.current',
        table='glbl.comune',
        formId='rec_gf_form',
        store_handler='record',
        storeType='Item')
    bar.dataController(
        "if(pkey){genro.formById('rec_gf_form').load({destPkey:pkey});}",
        pkey='^.comune_pkey')

The wiring is deliberately explicit: the selector writes to ^.comune_pkey, a dataController watches that path, and on every change it asks genro.formById('rec_gf_form') to load the record at that key. The form is the master of its own lifecycle — the outside world only triggers it.

Record store with resource

Same record-store shape, but the grouplet body comes from a table resource instead of an inline handler. Use this when the editor is reusable across pages and lives in the table's resource folder.

frame.center.contentPane(padding='10px').groupletform(
    resource='anagrafica',
    table='glbl.comune',
    formId='res_gf_form',
    store_handler='record',
    storeType='Item')
bar.dataController(
    "if(pkey){genro.formById('res_gf_form').load({destPkey:pkey});}",
    pkey='^.comune_pkey')

The resource name resolves under the table's resource directory; the rest of the wiring is identical to the handler version.

Nested resource path

Resources can live in subfolders. Pass a path with forward slashes:

frame.center.contentPane(padding='10px').groupletform(
    resource='territorio/altimetria',
    table='glbl.comune',
    formId='nested_gf_form',
    store_handler='record',
    storeType='Item')

Useful for organising several editors per table by topic — one file per editor, grouped in folders.

How to load a record programmatically

The form exposes its load entry point through the global registry. From any controller in the page:

genro.formById('rec_gf_form').load({destPkey: pkey});

The idiomatic trigger is a dataController listening to the selector's value — the same pattern shown in the record examples above. The same handle also gives you .save(), .reload(), and access to the form status (noItem, loaded, dirty, error).

Gotchas

formId must be unique within the page

Collisions break genro.formById() lookup: the second form registered with the same id silently shadows the first, and every external load/save call ends up driving the wrong one. The widget will auto-generate an id if you omit it, but auto-ids cannot be referenced from a controller — always set a stable, descriptive formId when you plan to drive the form from outside.

Memory store doesn't validate fields by default

The form lifecycle only handles load, save and dirty tracking; it does not run field-level rules. If a memory editor needs required fields or format checks, add validate_notnull (and friends) on individual widgets inside the inner grouplet — the form will surface those errors through its status, but it won't invent them for you.

Used by

groupletPanel and groupletWizard are built on top of groupletForm — each panel section and each wizard step instantiates a GroupletForm internally (see the center.GroupletForm(**grouplet_kwargs) calls in grouplet.py). Anything you learn about the form widget here carries over to those higher-level containers.
