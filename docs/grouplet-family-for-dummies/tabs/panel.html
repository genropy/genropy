groupletPanel

A navigation container that shows one grouplet at a time, with a tree or button-bar to switch between them.

When to use

You have a collection of grouplets — a topic folder, the grouplets directory of a table, or a custom grouplets_root — and you want the user to flip between them like sections of a settings page. groupletPanel wires that up for you: it builds the menu from the resource tree, renders a chrome around it (tree on the left or button bar at the top), and swaps the active grouplet into a center pane as the selection changes.

Internally each panel is mounted inside a GroupletForm, so every section keeps its own form status (clean / dirty / error) and its own save lifecycle. Switching to another section does not throw away the editing state of the previous one.

Two navigation modes

The same widget renders differently depending on whether you pass a topic. The dichotomy is intentional: a tree is right when you want the full map of sections, a button bar is right when there are few siblings inside a single topic.

Tree mode

No topic param. The full tree of grouplets (all topics, all leaves) is rendered on the left in a borderContainer with a splitter. Selecting a leaf swaps the right pane. Use when the user needs to see the whole map.

MultiButton mode

Pass topic='...'. The widget uses a framePane with a multibutton in the top slot bar, filtered to that single topic's children. Use when you have a handful of siblings inside one topic and a tree would feel over-engineered.

Tree mode example

From 07_grouplet_nested_topics.py, test_2_nested_topic_panel. No topic is passed, so the panel shows the whole grouplet tree of the page's package on the left and the selected grouplet (including expandable nested topics like submethodtesting/inner/) on the right.

def test_2_nested_topic_panel(self, pane):
    pane.borderContainer(height='500px', border='1px solid silver',
                         datapath='.nested_panel').groupletPanel(
        value='^.data',
        frameCode='nested_panel',
        region='center')

The panel reads its menu from gr_getGroupletMenu() with no filter, so every topic and grouplet found under resources/grouplets/ ends up in the tree. The frameCode is the identifier the panel uses for its internal frame and form — pick a unique one per panel on the same page.

MultiButton mode example

From 06_grouplet_navigation.py, test_2_panel_multibutton_table. Passing topic='technical' together with table='test.myticket' filters the menu to that table's grouplets/technical/ branch and flips the chrome to a button bar at the top:

def test_2_panel_multibutton_table(self, pane):
    pane.borderContainer(height='500px', border='1px solid silver',
                         datapath='.panel_mb').groupletPanel(
        table='test.myticket',
        topic='technical',
        value='^.data',
        frameCode='mb_table_panel',
        region='center')

The button bar shows one button per grouplet inside the technical topic. Clicking a button drives .selected_code, which the panel's controller turns into a resource swap on the center pane.

Without form wrapper (useForm=False)

By default the panel wraps each grouplet in a GroupletForm so the bound Bag has dirty tracking, autosave, status broadcasting. You can opt out with useForm=False when there is no per-panel editing — for example a read-only dashboard of sections, or a custom flow where you handle persistence yourself.

def test_3_panel_multibutton_no_table(self, pane):
    pane.borderContainer(height='500px', border='1px solid silver',
                         datapath='.panel_app').groupletPanel(
        topic='app',
        useForm=False,
        value='^.record',
        frameCode='mb_app_panel',
        region='center')

With useForm=False the center pane mounts a plain grouplet against the same value path. No form id, no autosave, no status pill on the bar — just the grouplet body.

Inside a Table Handler form

This is the canonical case: the form of a table (prospect, contact, ticket, ...) has a header with a few primary fields and a center area divided into many optional sections. Instead of stacking accordions, drop a groupletPanel rooted on ^#FORM.record (or a child path of it) and let the user pick the section.

From resources/tables/myprospect/th_myprospect.py:

class Form(BaseComponent):
    py_requires = 'gnrcomponents/grouplet/grouplet:GroupletHandler'

    def th_form(self, form):
        bc = form.center.borderContainer()
        top = bc.contentPane(region='top', datapath='.record')
        fb = top.formlet(cols=2, border_spacing='3px')
        fb.field('company_name', colspan=2, width='100%')
        fb.field('contact_name')
        fb.field('contact_email')
        # ... more header fields ...
        bc.contentPane(region='center').groupletPanel(
            value='^#FORM.record.extra_data',
            table='test.myprospect',
            frameCode='prospect_panel')

The header edits the columns of the record directly; the panel edits a Bag column (extra_data) sectioned by the grouplets under tables/_packages/test/myprospect/grouplets/. The whole structure lives inside the same TH form, so the standard Save / Cancel buttons commit everything at once.

Custom navigation menu

Sometimes the built-in tree or button bar is not what you want — you need a custom layout, an icon menu, a sidebar that mixes grouplet entries with other links. In that case, skip groupletPanel and call gr_getGroupletMenu directly to fetch the same menu Bag, then render it however you like and drive a plain grouplet with the selected resource.

def test_7_menu_tree(self, pane):
    bc = pane.borderContainer(height='500px', datapath='.comune_menu')
    left = bc.contentPane(region='left', width='220px', splitter=True)
    left.dataRpc('.grouplets_menu', self.gr_getGroupletMenu,
                 table='glbl.comune', _onStart=True)
    left.tree(storepath='.grouplets_menu',
              labelAttribute='caption',
              connect_onClick="""
                  if($2.item.attr.resource && $2.item.attr.grouplet_caption){
                      SET .selected_resource = $2.item.attr.resource;
                      SET .selected_caption = $2.item.attr.grouplet_caption;
                  }
              """)
    right = bc.borderContainer(region='center')
    right.contentPane(region='center').grouplet(
        resource='^.selected_resource',
        table='glbl.comune',
        value='^.data')

The helper gr_getGroupletMenu(table=..., topic=..., grouplets_root=...) is defined in resources/common/gnrcomponents/grouplet/grouplet.py and returns a Bag where each leaf carries resource and grouplet_caption attributes. That is enough to feed any widget that takes a storepath.

Parameters

ParamTypeDefaultDescription
valuestrrequiredData binding path. The selected grouplet reads and writes its Bag here.
tablestrNoneTable reference (e.g. 'test.myticket') for table-level grouplets. Resolves the menu under that table's grouplets/ directory.
topicstrNoneTopic name. If set, the panel uses multiButton mode filtered to that topic; otherwise tree mode.
frameCodestr'grplt_panel'Identifier for the internal frame and form. Use a unique value per panel on the same page.
useFormboolTrueWrap each grouplet in a GroupletForm with autosave store and status broadcasting.
useRecordPathboolFalseForce the form data path to .record instead of the dynamic location path of the selected grouplet.
menuCallbackcallableNoneCustom menu builder. Called as menuCallback(table, topic, grouplets_root), must return a menu Bag in the same shape as gr_getGroupletMenu.
grouplets_rootstr'grouplets'Custom root folder name under each package. Useful when one app exposes multiple unrelated grouplet sets (e.g. 'myconfigurations').
grouplet_*any—Extra kwargs prefixed with grouplet_ are forwarded to the underlying grouplet loader (the same convention as pane.grouplet).

Gotchas

Switching tabs doesn't auto-save

With useForm=True (the default) each panel keeps its own dirty state in its own GroupletForm. When the user clicks another section, the previous form is not saved automatically — it stays dirty in memory until something triggers a save (Save button, page close, explicit RPC). If you want auto-save-on-switch, wire a connect_onChange on each grouplet form that fires a save, or surface a status pill so the user notices unsaved work.

frameCode collisions

Two groupletPanel instances on the same page with the same frameCode will share the internal frame/form id and step on each other's status events. Always pass a unique frameCode per panel.
